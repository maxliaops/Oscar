Oscar News Archive

  • Oscar has been updated to 1.0.3 and is now available. This release represents mostly minor changes; see the change log for details. (January 21, 2005)
  • Oscar has been updated to 1.0.2 and is now available. This is more compliant with the specification than ever and should be downloaded; see the change log for details. (October 8, 2004)
  • Oscar source code is now available via Subversion; refer to the Oscar Subversion repository document for more information. (September 10, 2004)
  • Oscar has been updated to 1.0.1 and is now available. This releases introduces some bug fixes and should be downloaded; see the change log for details. (July 1, 2004)
  • Oscar 1.0 is now available! The 1.0 release introduces some significant changes, so please read the change log. (June 13, 2004)
  • Oscar is now hosted on ObjectWeb's forge site. (June 13, 2004)
  • The Oscar Bundle Repository web site is now online at Oscar's old web address. (June 13, 2004)
  • Updated the tutorial to work with version 0.9.4a of Oscar and to fix bugs in the Ant build file and example 7. (March 7, 2004)
  • A small update of Oscar, version 0.9.4a, is available that fixes a refresh problem for Windows users due to an unclosed file. It is probably a good idea for Linux users to upgrade too. See the change log for a complete list of changes. (November 27, 2003)
  • The final release of Oscar version 0.9.4 is available; it contains some important bug fixes for Windows users, but more importantly adds new functionality and compliance with the OSGi specification, including the Start Level service from OSGi 3. See the change log for a complete list of changes. (November 14, 2003)
  • Michel D'Hooge has made instructions available for using Eclipse and Oscar together; refer to the document for more details. (November 14, 2003)
  • A second pre-release of version 0.9.4 is available; it fixes many obscure bugs and a few not so obscure ones. This release should be stable and will be the last release before the final release of version 0.9.4. (September 1, 2003)
  • A pre-release of Oscar version 0.9.4 is now available; its version number is 0.9.4pre1. This pre-release represents a major re-engineering effort for Oscar. There is very little new functionality in this release, but many aspects of Oscar were affected. To see a brief summary of the changes refer to the change log. Please download this pre-release and send me reports of any issues you uncover. (June 25, 2003)
  • Yet another minor update to Oscar; the bug fix in the last release did not completely fix the class loading bug it was supposed to, but I am confident that this update finally eliminates the issue. See the change log for other minor changes; this release should be considered the most stable Oscar release. (June 16, 2003)
  • Another updated Oscar release that fixes some minor issues; people using versions 0.9.3 or 0.9.3a should upgrade. (May 26, 2003)
  • Updated Oscar release that fixes some very minor issues. (Apr. 14, 2003)
  • Major new release of Oscar that includes features from the OSGi 3.0 specification, such as dynamic package import. Oscar's license has also been changed to the Apache Software License. (Apr. 11, 2003)
  • The Device Manager implementation from Stephane Frenot is now available via the Oscar Bundle Repository (see the obr Oscar shell command); for more information refer to the documentation. (Apr. 4, 2003)
  • A new tutorial is online that demonstrates how to create bundles of varying complexity and also demonstrates how the Gravity Service Binder can simplify bundle development. (Jan. 21, 2003)
  • As of version 0.9.2, Oscar can download all example bundles directly off the Web! Contact me about adding your bundles to the Oscar bundle repository. (Nov. 27, 2002)
